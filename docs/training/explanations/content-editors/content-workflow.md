# Content Workflow

**Type:** Explanation  
**Audience:** Content Editors

## How Content Flows

### 1. Create Content

- Write Markdown files
- Add frontmatter
- Save to appropriate directory

### 2. Commit Changes

- Commit to Git
- Push to repository

### 3. Automatic Build

- GitHub Actions builds site
- Content validated
- Site generated

### 4. Deployment

- Built site deployed
- Live in 2-5 minutes

## Content Types

### Newsletters

Monthly updates in `src/pages/newsletters/`

### Resources

Informational pages in `src/pages/content/`

### Images

Media files in `public/images/`

### Booklets

PDFs in `public/booklets/`

## Workflow Best Practices

- Write content in Markdown
- Use proper frontmatter
- Test locally if possible
- Commit with clear messages
- Wait for deployment
